# Power BI – Transport Cost Analytics Dashboard
## Complete Setup Guide (Commits 10–15)

> **Note:** `.pbix` files are binary and cannot be auto-generated by a script.  
> Follow the steps below in Power BI Desktop to build the dashboard manually.  
> Each numbered section maps to one commit in the plan.

---

## Commit 10 – Connect to Database and Load Views

### Steps

1. Open **Power BI Desktop**
2. Click **Home → Get Data → More → Database → PostgreSQL database**
3. Enter connection details:
   - **Server:** `localhost` (or your server address)
   - **Database:** your database name
   - **Data Connectivity mode:** `Import`
4. Click **OK**, authenticate, then in the Navigator select these objects:
   - Tables: `carriers`, `routes`, `shipments`, `costs`, `delays`
   - Views: `vw_cost_per_km`, `vw_on_time_rate`, `vw_monthly_trends`, `vw_kpi_summary`
5. Click **Load**
6. Save as `powerbi/transport_dashboard.pbix`

## Commit 11 – Data Model and Relationships

### Star schema to build

In **Model view**, create the following relationships:

| From (Many) | Column | To (One) | Column | Cardinality |
|---|---|---|---|---|
| `shipments` | `carrier_id` | `carriers` | `carrier_id` | Many-to-one |
| `shipments` | `route_id`   | `routes`   | `route_id`   | Many-to-one |
| `costs`     | `shipment_id`| `shipments`| `shipment_id`| One-to-one  |
| `delays`    | `shipment_id`| `shipments`| `shipment_id`| Many-to-one |

> Views (`vw_*`) are already pre-joined — they connect to `shipments` via `shipment_id` if needed,  
> but primarily used as standalone reporting tables.

### Date table (recommended)

Create a Date table using New Table:
```dax
DateTable = 
CALENDAR(DATE(2023,1,1), DATE(2024,12,31))
```
Then add calculated columns:
```dax
Year   = YEAR(DateTable[Date])
Month  = MONTH(DateTable[Date])
MonthName = FORMAT(DateTable[Date], "MMM YYYY")
Quarter = "Q" & QUARTER(DateTable[Date]) & " " & YEAR(DateTable[Date])
```
Connect `shipments[shipment_date]` → `DateTable[Date]`.

## Commit 12 – DAX Measures (KPIs)

Create a dedicated **Measures** table (New Table → `Measures = {0}`, then hide the column).

Add these measures to the Measures table:

### Core Cost Measures

```dax
Total Cost = 
SUM(costs[total_cost])

Avg Cost per Shipment = 
DIVIDE([Total Cost], [Shipment Count])

Avg Cost per KM = 
DIVIDE(
    SUM(costs[total_cost]),
    SUMX(RELATEDTABLE(routes), routes[distance_km])
)

Total Fuel Cost = SUM(costs[fuel_cost])
Total Toll Cost = SUM(costs[toll_cost])
Total Labour Cost = SUM(costs[labour_cost])
```

### Volume Measures

```dax
Shipment Count = 
COUNTROWS(FILTER(shipments, shipments[status] <> "cancelled"))

Total Weight KG = 
SUM(shipments[weight_kg])
```

### Delivery Performance Measures

```dax
Delivered Count = 
CALCULATE(COUNTROWS(shipments), shipments[status] = "delivered")

On-Time Count = 
CALCULATE(
    COUNTROWS(shipments),
    shipments[status] = "delivered",
    shipments[actual_delivery] <= shipments[expected_delivery]
)

On-Time Rate % = 
DIVIDE([On-Time Count], [Delivered Count], 0) * 100

Late Count = 
CALCULATE(
    COUNTROWS(shipments),
    shipments[status] = "delivered",
    shipments[actual_delivery] > shipments[expected_delivery]
)

Delay Rate % = 
DIVIDE([Late Count], [Delivered Count], 0) * 100

Avg Days Late = 
AVERAGEX(
    FILTER(
        shipments,
        shipments[status] = "delivered" &&
        shipments[actual_delivery] > shipments[expected_delivery]
    ),
    shipments[actual_delivery] - shipments[expected_delivery]
)
```

### MoM Change Measure

```dax
Cost MoM Change % = 
VAR CurrentMonth = [Total Cost]
VAR PreviousMonth = CALCULATE(
    [Total Cost],
    DATEADD(DateTable[Date], -1, MONTH)
)
RETURN
DIVIDE(CurrentMonth - PreviousMonth, PreviousMonth, BLANK()) * 100
```

## Commit 13 – Dashboard Page 1: Cost Overview

### Page name: `Cost Overview`

**Layout:**

```
┌─────────────────────────────────────────────────────────┐
│  [KPI] Total Cost    [KPI] Avg Cost/KM  [KPI] Shipments │
├─────────────────────────┬───────────────────────────────┤
│  Bar chart:             │  Line chart:                   │
│  Total Cost by Carrier  │  Monthly Cost Trend (2023-24)  │
│  (carriers[name] on X)  │  (DateTable[MonthName] on X)   │
├─────────────────────────┴───────────────────────────────┤
│  Table: Top Routes by Cost                              │
│  Columns: Origin → Dest | Distance | Shipments | Cost  │
│  Sorted: Total Cost DESC                                 │
├─────────────────────────────────────────────────────────┤
│  Donut: Cost breakdown (Fuel / Toll / Labour / Other)   │
└─────────────────────────────────────────────────────────┘
```

**Visual settings:**
- KPI cards: bold white text, colored background (blue #1E5F9B)
- Bar chart: single color #2196F3, data labels on
- Line chart: smooth line, markers on, area fill with 20% opacity
- Slicer: `DateTable[Year]` (top-right)
- Slicer: `carriers[carrier_type]` (top-right)

## Commit 15 – Forecast and Final Styling

### Forecast

1. On Page 1, select the Monthly Cost Trend line chart
2. In **Visualizations → Analytics panel** enable:
   - **Forecast**: Periods = 6, Confidence interval = 95%
   - **Trend line** (for historical context)

### Color Theme

Apply a consistent theme via **View → Themes → Customize theme**:

```json
{
  "name": "Transport Analytics",
  "dataColors": [
    "#1E5F9B", "#2196F3", "#4CAF50", "#FF9800",
    "#F44336", "#9C27B0", "#00BCD4", "#607D8B"
  ],
  "background": "#FFFFFF",
  "foreground": "#333333",
  "tableAccent": "#1E5F9B"
}
```

### Tooltips

For each bar/line chart, add a custom tooltip page:
- Page name: `Tooltip_Cost`
- Add: avg cost/km, carrier type, shipment count
- Set page type to **Tooltip** in Page settings
